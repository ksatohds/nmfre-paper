> **Wild Bootstrap Inference for Non-Negative Matrix Factorization with Random Effects**
> Kenichi Satoh (Shiga University)
> *Psychometrika* Special Issue on Variable Selection for Complex Psychometric Data (2026)

### R version
- R >= 4.3.0 (developed and tested on R 4.5.2)
- Linux recommended for parallel computation (`mclapply` uses fork)
- Windows: simulations run serially (still produces identical results)

### Important: quanteda version and corpus data

The `data_corpus_inaugural` dataset in `quanteda` may change between versions
(e.g., new inaugural addresses added). The results in the paper were produced
with **quanteda 4.2.0** (59 documents, Washington 1789 -- Biden 2021).

To ensure exact reproducibility, run `save_inaugural.R` **once** to save
the corpus locally:

```r
# Step 1: Check quanteda version
packageVersion("quanteda")

# Step 2: Save corpus locally
source("save_inaugural.R")
# -> creates data/inaugural_corpus.RData
```

The analysis and STM benchmark scripts will automatically load from
`data/inaugural_corpus.RData` if it exists, falling back to
`quanteda::data_corpus_inaugural` with a warning otherwise.

## How to Reproduce

### Quick start (benchmark scripts)
These scripts run in seconds and require minimal dependencies:
```r
rmarkdown::render("benchmark_lmm.rmd")    # §4.1 LMM
rmarkdown::render("benchmark_mimic.rmd")   # Table 6 MIMIC
rmarkdown::render("benchmark_ppr.rmd")     # §4.3 PPR
rmarkdown::render("benchmark_stm.rmd")     # Table 9 STM
```

### Empirical illustrations (~10 minutes)
Produces Tables 4, 5, 7, 8 and Figures 1–4:
```r
rmarkdown::render("analysis_empirical.rmd")
```
Requires: `nmfkc`, `nlme`, `lavaan`, `quanteda`, `NipponMap`,
`rsvg`, `DiagrammeRsvg`, `httr`, `readxl`

### Simulations (computationally intensive)

**Baseline simulation** (Table 1 / Table F1):
- 2 error distributions × 2 hypotheses × 3 sample sizes × 1000 replicates
- ~2–3 hours on 24-core Linux server
```r
rmarkdown::render("sim_baseline.rmd")
# Output: sim_summary_baseline.csv
```

**Stress test simulation** (Table 2 / Table E1):
- 2 error distributions × 2 hypotheses × 3 cap settings × 1000 replicates
- ~3–4 hours on 24-core Linux server
```r
rmarkdown::render("sim_stress.rmd")
# Output: sim_summary_dfUcap_stress.csv
```

## NMF-RE Algorithm

The core algorithm is in `nmfre_core.R`, providing:

- `nmf.re()`: Main estimation function
  - Multiplicative non-negative updates for X and Θ
  - Ridge BLUP updates for U with dfU cap enforcement
  - Wild bootstrap inference (exp-centered weights)
  - One-step Newton bootstrap replicates projected to Θ ≥ 0
- `summary.nmfre()`: S3 summary method
- `scan_dfU_cap_rate()`: Diagnostic scan over dfU cap rates
- Internal helpers: `normalize_X()`, `.nndsvdar()`, `.dfU_from_lambda()`, `.lambda_for_dfU_cap()`

A user-friendly R package wrapper is available at:
https://github.com/ksatohds/nmfkc

## Reproducibility Notes

- **Deterministic seeding**: Per-replicate seeds (`set.seed(base + sim_id + 100000*N)`)
  ensure identical results regardless of parallelization platform
- **Platform verified**: All results reproduced on both Windows 11 + R 4.4.2
  and Ubuntu 24.04 + R 4.5.2 (OpenBLAS)
- **Simulation results**: Baseline simulation (Table F1) matches exactly at
  3 decimal places across platforms due to per-replicate seeding
